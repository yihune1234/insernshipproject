# Frontend and Mobile Integration Agent Prompt

## Objective

Integrate the backend with both web frontend and mobile applications.

The backend is the single source of API services.

Frontend and mobile applications must consume the same APIs.

The integration must support:

* Web application
* Mobile application
* Offline mobile capabilities
* Credential synchronization
* Wallet management
* Verification
* Notifications

---

# General Rules

* Do not create separate business logic in frontend or mobile.
* All business rules must remain in the backend.
* Frontend and mobile only consume APIs.
* Use the same API contracts for both clients.
* Every API response must follow the common response format.
* All endpoints must be documented.

---

# Backend API Requirements

Every endpoint must provide:

* status
* message
* data
* errors
* pagination (if needed)

Example:

```json
{
    "success": true,
    "message": "Success",
    "data": {}
}
```

---

# Authentication Integration

Both frontend and mobile must support:

* Register
* Login
* Refresh token
* Logout
* OTP verification
* Session validation

Required endpoints:

```text
POST /auth/register
POST /auth/login
POST /auth/refresh
POST /auth/logout
GET  /profile
```

---

# Mobile Authentication Rules

Mobile application must support:

### Online Login

* authenticate with server
* obtain JWT tokens

### Offline Login

Allow access using:

* local encrypted session
* biometric authentication
* device authentication

Conditions:

* user previously logged in
* session not expired
* local wallet available

Offline mode must not perform:

* synchronization
* verification against organizations
* remote API calls

---

# Organization Integration APIs

Frontend and mobile must access:

```text
GET  /organizations
POST /organizations/register
GET  /organizations/{id}
```

Features:

* organization selection
* organization connection
* integration status

---

# Credential Synchronization APIs

```text
POST /credentials/sync
GET  /credentials
GET  /credentials/{id}
```

Flow:

User selects organization.

↓

User requests synchronization.

↓

Backend validates holder.

↓

Backend synchronizes credentials.

↓

Wallet updated.

---
